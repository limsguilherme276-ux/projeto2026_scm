Gerenciamento de Configuração de Software (SCM) 🚀Este projeto foi desenvolvido como um Relatório de Aula Prática para a disciplina de Projeto de Software na Faculdade Anhanguera (Polo São João de Meriti - RJ). O foco principal foi a aplicação de conceitos de Gerenciamento de Configuração de Software (SCM) em um cenário real de desenvolvimento.🎯 ObjetivosImplementar o controle de versão utilizando Git e GitHub.Praticar a integração entre o ambiente de desenvolvimento local (VS Code) e o repositório remoto.Garantir a integridade e segurança dos artefatos de software através de comandos de configuração.🛠️ Tecnologias e FerramentasGit: Controle de versão e gerenciamento de histórico.GitHub: Hospedagem do repositório remoto oficial (projeto2025_SCM).Visual Studio Code (VS Code): IDE utilizada para codificação e operações de terminal.Python: Linguagem utilizada para a criação do arquivo de teste novo.py.📈 Fluxo de Trabalho RealizadoDurante a atividade, realizei as seguintes etapas fundamentais de SCM:Criação do Repositório: Configuração do ambiente remoto no GitHub.Clonagem Local: Uso do comando git clone para baixar o projeto e iniciar o desenvolvimento no VS Code.Ciclo de Versão:Criação e edição do arquivo novo.py.Uso de git add para preparar as alterações.Registro de mudanças com git commit, garantindo mensagens claras sobre a evolução do código.Sincronização e Status: Monitoramento constante via git status para assegurar que a árvore de trabalho estivesse limpa e atualizada com a versão "oficial".🧠 Conclusão e AprendizadosA atividade reforçou que o uso correto do GitHub é essencial para evitar conflitos em times profissionais, garantindo que sempre exista uma versão centralizada e segura do projeto. Desenvolvi habilidades práticas com os comandos add, commit e push, fundamentais para a organização de qualquer projeto de software.Informações Acadêmicas:Aluno: Guilherme Gomes de Lima Instituição: Faculdade Anhanguera Data: 02/03/2026
